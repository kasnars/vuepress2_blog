1. TreeShaking原理，是基于什么实现的，有什么作用？

```
原理：通过静态代码分析，做到依赖关系追踪，未引用代码的检测，代码剪裁
实现方式：基于es6模块系统实现，转为AST进行解析
作用：减小包的体积，优化依赖管理（清理第三方代码库中未被使用的模块），提升代码质量

注意事项：某些代码需要通过siedEffacts 来标记是否有副作用，以防止误删

```

1. Vue2和Vue3有哪些区别，Vue3做了哪些升级？

```
响应式实现方式升级：Object.defineProperty -> Proxy
Api组织形式：Options Api -> Composition Api
性能优化：虚拟dom性能提升，TreeShaking，编译优化（静态节点标记和预编译）

新功能：Fragment，Teleport（允许部分内容渲染到DOM任意位置，例如body），Suspense（异步组件加载），ref，reactive，watchEffact，原生ts支持

总结：性能更强，代码更灵活，生态更现代

```

1. Vue3 composition Api和原先的有什么区别？有哪些优点？

```
通过hook的方式使逻辑服用更高效

结合ref使响应式更加灵活
对ts支持更友好
逻辑更易于拆分，易于treeshaking

```

1. JS有哪些常用的设计模式，分别有哪些作用？

```
工厂模式：统一接口创建对象，解耦对象实例化过程
单例模式：确保一个类只有一个实例，全局共享
建造者模式：分步骤构建复杂对象，隔离构建细节

适配器模式：转换接口，使不兼容的对象可以协同工作
装饰器模式：动态扩展对象功能，不修改源代码
代理模式：控制对象访问，实现缓存权限等（Proxy）

观察者模式：对象状态变化时通知依赖他的对象（发布-订阅）
策略模式：在运行时选择不同算法，避免大量条件判断
模板方法模式：定义算法骨架，子类实现具体步骤

享元模式：共享对象减少内存消耗（如缓存DOM节点）
职责链模式：请求在多个处理器之间传递，直到最后被处理（例如中间件）
命令模式：将请求封装为对象，支持撤销，重做等操作

```

1. 防抖和节流有什么区别？搜索框用哪种？

```
防抖是B回城，中间干啥都会被打断；事件触发后延迟一段时间执行
节流是Q技能，cd好了就能用，用了等下一次cd；规定的时间间隔内只能使用一次

防抖用处：用户操作结束后再准确响应：搜索框输入，窗口大小调整后重新计算
节流用处：限制事件触发的频率：页面滚动，鼠标移动

```

1. Class有哪些特性？

```
封装：将数据和操作方法封装在一个单元中，隐藏内部实现细节
继承：一个类可以继承另一个类的属性和方法
多态：不同类对同一个消息做出不同反应，比如dog和cat都继承animal，都有sound的方法，但是dog和cat重写了方法，实现不同的效果
抽象：定义抽象类和抽象方法，不能被实例化，只能作为其他类的的基类，方法只有声明没有实现，比如图形类，里面有面积这个方法，但是具体实现要三角形，正方形这些子类自己去实现
实例化：类是对象的模板，可以创建多个实例对象，每个对象有自己的属性值，但是共享类的方法

```

1. 说说es6的Map和Set是什么

```
Map 键值对 get set has size
Set 集合 不包含重复元素 add delete has size

map与weakmap的区别：
weakmap的key只能用对象，并且是弱引用会被自动回收，不支持遍历，不确定条目，无size属性，weakmap性能更好，且内存泄漏风险更低，适合key的对象生命周期不确定的场景，dom操作，私有属性等

```

1. 面向过程是什么？面向对象有哪些特性，怎么理解？

```
面向过程：
强调步骤和顺序，数据与操作分离，自顶向下设计

面向对象三大特性：
封装：数据和操作封装在一起，对外部隐藏细节，只提供对外的接口
继承：通过子类继承父类的属性和方法实现复用，同时也可以重写
多态：一个方法在不同对象上调用时出现不同的行为，通过方法重写和接口来实现

```

1. 防抖节流有什么区别，一般什么时候用

```
防抖：触发后倒计时，中间触发重新计时：用户不动了再响应，搜索框，窗口缩放
节流：固定间隔时间内触发一次，再触发不反应：避免多次请求：滚动加载，鼠标移动

```

1. 单例模式和工厂模式是什么？工厂模式一般用来干什么？

```
单例模式：一个类一个实例，全局访问点，任务管理器，任何地方调用都是同一个实例
工厂模式：创建对象，将对象和使用进行分离，侧重对象的创建和管理，例如造车通过传入不同的参数来决定生产什么车

前端中的工厂模式：创建不同类型组件，返回动态组件，后台返回的数据格式化为一个特定的格式，重点是返回对象

```
